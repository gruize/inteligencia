Palillos
  • Descripción del juego:

    Disponemos de 6 palillos y 2 adversarios. El juego consiste en ir retirando en cada turno 1 o 2 palillos del montón, y ganará el adversario que consiga quitar todos los palillos. En esta implementación añadimos la condición de que gané el jugador inicial.

  • Heurística

Aplicamos un algoritmo no informado o ciego, pero el juego está preparado para realizar busquedas informadas.

h'(n) = Número de palillos restantes, si el jugador es el 0.

h'(n) = Número de palillos restantes + 1, si el jugador es el 1.

  • Operadores

  1. Quitar un palillo: Si y sólo si el numero de palillos restantes es mayor o igual que 1
  2. Quitar dos palillos: Si y sólo si el numero de palillos restantes es mayor o igual que 2

  • Algoritmo

Algoritmo IterativeDeepeningSearch o Profundidad Iterativa: Combina los algoritmos de Primero en Anchura y Primero en Profundidad, donde la profundidad se desconoce, ya que ésta varía de forma creciente.

 

  • Espacio de estados

Estado ( Palillos restantes, Jugador )
     Palillos restantes = {0, 1, 2, 3, 4, 5, 6}
     Jugador = {0, 1}

Estados prohibidos:
     Estado( 6, 1 )
     Estado( 0, 1 )

Estos estados prohibidos se generan debido a las especificaciones añadidas al juego (Gana el jugador inicial y éste es el Jugador 0)

Estado inicial:
     Estado( 6, 0 )

Estado final:
     Estado( 0, 0 )
